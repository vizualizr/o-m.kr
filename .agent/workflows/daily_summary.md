# Daily Summary & Memory Sync Workflow

이 워크플로우는 하루 작업을 마무리할 때 실행하며, 다음 5개 단계를 순차적으로 수행합니다.
별도 지시가 없다면 하루의 기준은 해당일 오전 5시부터 익일 오전 2시까지이다. 만약 이 명령을 2020년 10월 14일 오전 1시에 수행한다면 대상 작업은 2020년 10월 13일 오전 5시부터 2020년 10월 14일 명령을 받은 시점까지 수행한 작업이다

## 1. 계획 문서 업데이트
- 오늘 변경된 사항 중 `.agent\usr\plans` 폴더의 문서들에 반영해야 할 핵심 내용이 있는지 검토합니다.
- 변경이 필요한 경우 사용자에게 확인을 받은 후 업데이트를 수행합니다.

## 2. 에이전트 전용 작업 로그 작성
- 다음 작업을 즉시 재개할 수 있도록 에이전트(나)에게 최적화된 상세 작업 현황을 기록합니다.
- **경로**: `.agent\daily-logs\YYYY_MM_DD.md`

## 3. Logseq 저널 생성
- `/logseq_format_summary` 워크플로우를 호출하여 저널 내용을 생성합니다.
- 사용자가 직접 확인하는 용도이므로 미시적인 코드 변경보다 **프로젝트 전체의 구성과 흐름** 중심으로 작성합니다.
- 결과물은 코드 블록 형식이 아닌 아래 경로의 파일로 직접 저장합니다.
- **경로**: `d:\yonggeun\porter\git\o-m.kr\journal\journals\YYYY_MM_DD.md`

// turbo
## 4. 맥락 및 메모리 백업 (Sync)
- 에이전트가 생성한 아티클(Task, Implementation Plan 등)과 메모리를 로컬 프로젝트와 동기화합니다.
- **실행**: `/backup_brain` 워크플로우를 호출하여 수행합니다.

// turbo
## 5. 로컬 Git 커밋
- 모든 변경 사항을 스테이징하고 커밋을 수행합니다.
- **커밋 메시지 자동 생성**:
  - 제목: 오늘 작업의 핵심 요약 (예: "feat: 그리드 시스템 리팩토링 및 워크플로우 추가")
  - 본문: 
    - [에이전트 로그 요약]: Step 2에서 작성한 로직 기반의 상세 변경 사항.
    - [사용자 저널 요약]: Step 3에서 작성한 프로젝트 흐름 중심의 요약 내용.
