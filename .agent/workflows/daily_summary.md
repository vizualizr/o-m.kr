# Daily Summary & Memory Sync Workflow

이 워크플로우는 하루 작업을 마무리할 때 실행하며, 다음 단계를 순차적으로 수행합니다.

## 0. 작업 범위 및 경로 제안 (Pre-check)
- 실행 전, 오늘 요약할 작업의 시간 범위와 결과물이 저장될 **절대 경로**를 사용자에게 보고합니다.
- 사용자의 "확인" 또는 "시작" 사인을 받은 후에만 1단계로 진행합니다.

## 1. 계획 문서 업데이트
- 오늘 변경된 사항 중 `.agent\usr\plans` 폴더의 문서들에 반영해야 할 핵심 내용이 있는지 검토합니다.

## 2. 에이전트 전용 작업 로그 작성
- 다음 작업을 즉시 재개할 수 있도록 에이전트(나)에게 최적화된 상세 작업 현황을 기록합니다.
- **경로**: `.agent\daily-logs\YYYY_MM_DD.md`

## 3. Logseq 저널 생성
- `/logseq_format_summary` 워크플로우를 호출하여 저널 내용을 생성합니다.
- **경로**: `d:\yonggeun\porter\git\o-m.kr\journal\journals\YYYY_MM_DD.md` (반드시 이 절대 경로를 준수할 것)

// turbo
## 4. 맥락 및 메모리 백업 (Sync)
- 에이전트가 생성한 아티클(Task, Implementation Plan 등)과 메모리를 로컬 프로젝트와 동기화합니다.
- **실행**: `/backup_brain` 워크플로우를 호출하여 수행합니다.

// turbo-all
## 5. 보고 및 일괄 승인 (Batch Approval)
- 1~4단계를 통해 준비된 **[에이전트 로그]**와 **[저널 내용]**을 요약하여 사용자에게 보고합니다.
- 사용자가 내용 확인 후 **"승인"** 또는 **"Go"**를 입력하면 다음 작업을 일괄 수행합니다.
  1. 물리적 파일 저장 (로그 및 저널)
  2. 로컬 Git 스테이징 및 커밋 수행

## 6. 로컬 Git 커밋
- 커밋 메시지 자동 생성:
  - 제목: 오늘 작업의 핵심 요약
  - 본문: [에이전트 로그 요약] 및 [사용자 저널 요약]

---

> [!IMPORTANT]
> 에이전트는 4단계까지의 준비 과정을 마친 후 사용자에게 최종 보고를 수행해야 하며, 5단계의 물리적 파일 쓰기와 Git 커밋은 반드시 사용자의 명시적 승인 후에만 실행한다.
