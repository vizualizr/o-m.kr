# Logseq Formatting Workflow

- 이 워크플로우는 사용자가 지시하는 특정 내용을 에이전트가 요약, 정리할 때 수행하는 표준 절차이다.
- 사용자가 다음에 해당하거나 유사한 요청을 전달하면 이 워크플로우를 수행한다.
- > ~를 마크다운으로 정리해
  > ~를 마크다운 서식(또는 형식)으로 정리해
  > ~를 록식(또는 logseq) 형식으로 정리해
  > ~를 록식(또는 logseq) 서식으로 정리해


## 1. 지침 및 템플릿 확인

- `.agent\usr\templates\logseq.md` 파일을 참조하여 현재 유효한 서식과 속성(properties) 정의를 확인합니다.

## 2. 서식 규칙

- 사용자가 지시하는 내용이 소프트웨어라면 템플릿에 있는 `소프트웨어 서식`을 이외에는 `일반 서식`을 사용한다.

### 공통 필수 규칙

- 대상이 되는 대화 내용이나 작업 내역을 다음 항목에 맞춰 정리한다.
    - **Summary**: 전체적인 작업의 목적과 결과.
    - **Steps**: 구체적인 수행 혹은 처리 단계.
    - **Troubleshooting**: 해당 내용에서 논의했거나 발생한 문제와 해결 방법.
    - **log**: 타임라인 기반의 주요 기록.
    - **References**: 참고한 문서나 링크.

- 문체는 `-다` 형식을 사용하며, 최대한 간결하게 개조식으로 작성한다.
- 마크다운 형식의 링크가 참조하는 대상의 경로가 로컬 자원(예: file:///...)일 경우 다음 규칙을 다른다.
  - 해당 경로가 현재 프로젝트 하위 경로(file:///d:/yonggeun/porter/git/o-m.kr/production/)이면 상대 경로를 사용할 경우 해당 하위 경로를 `https://github.com/vizualizr/o-m.kr/blob/main/`로 변경한다.

### 일반 서식

- division은 기준에 따라 다음 가운데 하나를 선택한다. 
  - [[makr]] - 프로그래밍, audiovisual authoring, 개발 업무 등을 포함해 가시적 성과를 만드는 일 일체. 단 글쓰는 일은 제외한다.
  - [[carta]] - 문자를 읽고 쓰는 일로 한정한다. 책 읽고 글 쓰는 일이 이에 해당한다.
  - [[mining]] - 생계를 위해 돈 버는 일이면 모두 포함한다.
  - stack은 개발 직무를 기준으로 frontend, backend, network 및 기타 이와 같은 계층의 분류를 기재한다. 개발 이외의 직무에는 웹 서비스 제작 및 유통에 해당하는 직무(예, design, marketing)를 기재할 수 있다.
- tag 속성을 기재할 때는 모두 소문자로 작성하며 에이전트가 판단해 기재한다. 단 검색 편의성을 위해 최소한의 해당 태그만 기입한다.
- type에는 아무런 내용도 넣어서는 안 된다.

### 소프트웨어 서식

- status는 현재 현황에 따라, [[incumbent]], [[installed]], [[pooled]], [[uninstalled]] 가운데 하나를 선택한다. 일반 정보를 기록한다면 [[pooled]]에 해당한다. 단순 설치 후 활용 전이면 [[installed]], 설치 후 사용자의 특정 작업에서 명시적 역할을 담당하면 [[incumbent]], 설치했으나 기타 사유로 더 이상 사용하지 않으면 [[uninstalled]]로 기재한다.
- host는 내가 소유한 아래 기기 가운데 하나를 기재한다. 최신 기기 현황은 `d:\yonggeun\.agent\workflows\a. assets\devices.md`을 기준으로 한다.

## 3. 최종 결과 출력
- 결과물을 사용자가 바로 복사하여 Logseq에 붙여넣을 수 있도록 **마크다운 코드 블록(```markdown ... ```)** 형식으로 제시합니다.
