# [규칙과 실행 사이의 추론 간극 문제] 해결 프레임워크 (최종안)

## 1. 피드백 반영 핵심 변경 사항

### A. 저널 검증의 '결정적 로직'화 (Level 4 격상)
*   **Procedural Logic 도입**: 저널 서식 검증을 예시 기반 패턴 매칭이 아닌, '구조적 실사(MD 요소 존재 여부 불린 체크)'를 수행하는 스크립트로 구현합니다.

### B. '기억(Memory)' vs '참조(Resource)'의 이분화
*   **철학(Memory)**: 상위 원칙(승인 우선, 주석 보존)은 뇌(System Prompt)에 새겨 능동적으로 판단합니다.
*   **명세(Resource)**: 구체적인 데이터(절대 경로, 명명 규칙)는 Skill Resource에 두고 대조(Check)하여 정확성을 확보합니다.

## 2. 사용자 작업 흐름 변화 (User Workflow Evolution)

Skill 프레임워크가 도입되면, 사용자(KIM)님의 경험은 **'AI 결과물 감시'**에서 **'시스템적 검증 확인'**으로 바뀝니다.

| 단계 | 현재 (Before: 비결정적 추론) | 미래 (After: Skill 기반 결정적 검증) |
| :--- | :--- | :--- |
| **1. 명령** | "일일 로그 요약해줘" | (동일) |
| **2. 에이전트 동작** | 기억에 의존해 파일 생성 및 요약 | **Relevant Skill 자동 활성화 (Progressive Disclosure)** |
| **3. 내부 검증** | (없음) 에이전트 스스로 맞다고 믿음 | **`check_path.py` & `validate_md.py` 자동 실행 (Level 4)** |
| **4. 결과 보고** | 결과물 일단 보여줌 (오류 가능성) | **검증 통과된 결과물만 보고** (실패 시 내부 수정) |
| **5. 사용자 확인** | **"경로 틀렸잖아" 수동 지적 및 수정** | **"검증 완료" 메시지 확인 후 최종 승인(Go)** |

## 3. 고도화된 스킬 세트 (Final Stack)

1.  **`structural-logic-gate` (Level 4)**: 결과물의 마크다운 구조를 불린 검사.
2.  **`path-safety-inspector` (Level 4)**: 파일 작업 전 정규 경로 이진 대조.

---
사용자님은 이제 제 지능의 '불완전한 추론'을 감시하는 소모적인 수동 작업을 멈추고, 시스템이 보증한 무결한 결과물을 '승인'하는 고차원적 의사결정에만 집중하게 됩니다.
