# o-m.kr/production guideline

- 해당 프로젝트의 입출력 언어는 한국어이다.
- 특정 파일에 변경 사항이 있을 경우 변수/함수 이름/설정/입출력 자료형과 숫자 등 다른 파일과 공유하는 부분을 반드시 대조 확인하여 사용자가 변경한 내용에 따라 관련 사항을 갱신한다.
- 사용자가 `오늘 하루 작업을 정리해 줘` 혹은 유사한 내용의 말로 하루 작업을 정리하고자 하면 아래 나열한 작업을 순서대로 진행할 것인지를 목록을 보여주며 사용자에게 최종 확인을 받은 후 각 작업을 진행한다. 최종 확인 과정에서 사용자가 요청하는 부가 사항을 반영해야 한다.
  1. 해당 일자에 변경된 사항 가운데 `.agent\usr\plans` 폴더에 반영해야 할 내용이 있다면 이를 수행한다.
  2. 해당 일자에 작업한 내역을 아래 경로에 마크다운 형식으로 저장한다. 별도의 에이전트 메모리 확인없이 해당 내역을 확인하는 것만으로 향후 작업을 바로 시작할 수 있도록 에이전트에게 최적화된 형태로 작업 현황을 기록한다.
  3. `.agent\usr\templates\logseq.md`에 기재된 지시사항과 서식에 따라 해당 내용을 마크다운 형식으로 작성해 `d:\yonggeun\porter\git\o-m.kr\journal\journals\`폴더에 `YYYY_MM_DD.md`라는 이름 규칙에 따라 저장한다. 사용자가 직접 복기하는 파일이므로 세부적인 변경 사항보다 프로젝트 전체의 구성과 흐름을 중심으로 작성한다.
  4. 해당 일자 작업 내역과 관련된 맥락 및 메모리의 동기화 및 백업을 수행한다. (.agent\workflows\backup_brain.md)
  5. 로컬 깃 저장소에 변경 사항을 커밋한다. 커밋에는 항상 커밋 로그를 첨부해야 한다. 커밋 로그는 간단히 작성하고 에이전트를 위한 세부사항은 2번 항목에 사용자를 위한 세부사항은 3번 항목에 기재한다.
- 사용자가 특정한 내용을 `마크다운 형식`으로 혹은 `록식 형식`으로 정리하라고 요청할 경우 `.agent\usr\templates\logseq.md`에 기재된 지시사항과 서식에 따라 해당 내용을 마크다운 형식으로 작성해 사용자가 바로 복사해서 붙여넣기 할 수 있도록 코드블록으로 제시한다.
- 코드 변경 후 브라우저에서 눈으로 확인이 필요한 사항은 에이전트가 브라우저 확장을 이용해 확인하지 않는다. 대신 사용자에게 변경 사항을 정리하면 사용자가 직접 눈으로 확인 후 에이전트와 추가 수정 사항 및 보완 사항을 논의 및 합의한다.
- 에이전트가 직접 생성하는 자바스크립트 파일은 해당 파일에서 데이터의 입력, 처리, 출력 흐름을 최대한 빨리 파악할 수 있도록 주석을 추가한다.
