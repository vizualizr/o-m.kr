4. 웹사이트의 지속적인 운영을 위한 마케팅 관련 기술 대응 및 최적화, 그리고 수익 창출 모델 모색
5. 위 목적에 부합하는 한 AI 에이전트는 이 문서 및 모든 관련 문서의 내용 일체를 자신의 맥락에 반드시 포함해야 한다.
6. 위 목적에 부합하는 한 AI 에이전트는 사용자의 지시 사항을 끝까지 이행하는 동시에 사용자의 지식 수준 향상에 기여해야 한다.
7. AI 에이전트는 이 프로젝트 및 관련 결과물에 해가 되는 제안, 결정, 작업 및 일체의 행위를 절대 금한다.

## 프로젝트 개요

### 프로젝트의 목적

o-m.kr 프로젝트는 독립형 데이터 저널리즘을 표방한다. 데이터 저널리즘이란 데이터와 통계를 바탕으로 사실을 전달하고 의견을 밝히는 방법이다. 독립형이라 함은 운영자인 김용근이 프로젝트의 총괄 및 사이트와 콘텐츠의 기획, 디자인, 개발, 콘텐츠 작성, 마케팅을 책임지는 1인 프로젝트임을 말한다. 

### 프로젝트 운영 전략

#### 수익화 전략

수익 구조를 이분화하여 운영과 성장을 분리한다. 온라인 광고로 운영 수익을 창출하며 분기마다 구독 수익을 점검해 성장을 위해 투자한다. 운영 부문의 주요 지출은 인프라 유지와 콘텐츠 유지 관리 및 생성이다. 성장 부문의 주요 투자처는 인프라 개선과 콘텐츠 제작을 위한 재교육 및 인프라 개선이다.

수익 창출을 위한 현황 및 3개월 단위의 세부 계획은 아래 문서를 참조한다.
`@./usr/plans/monetization.md
`
#### 콘텐츠 전략

다음을 전제로 한다.

- 오늘날 텍스트 기반 저널리즘의 수익 모델은 닷텀 버블 직후의 모델에 여전히 정체되어 있다.
- 텍스트 기반 미디어의 주 소비층은 여전히 활자 독해의 장점을 인정하는 소수의 고학력층이다.
- 그러나 이들조차 자신들이 어떤 형태의 뉴스 미디어를 원하는지 모른다.
- 분명한 점은 텍스트 기반 저널리즘의 수익 모델이 더 저널리즘의 발전을 저해하고 있다는 점이다.
 - 클릭 기반 수익 모델은 텍스트의 가치를 절하하며,
 - 구독 모델은 산업 전체의 퇴행이다.
- 따라서 우리는 다음을 모색한다.
 - 텍스트 기반 저널리즘의 새로운 형식.
 - 텍스트 기반 저널리즘의 생명을 연장할 수 있는 새로운 기술적 방법.
 - 구글이 정의한 인터넷 광고 모델에서 텍스트의 가치를 재정립할 수 있는 방법을 모색한다.

본 프로젝트의 콘텐츠는 위에서 제시한 문제를 해결할 수 있는 대안을 찾기 위해 다양한 방법을 실험한다.

개발 노트 및 문서화는 로컬 Logseq에서 작성되며, Github Action(logseq-SPA)을 통해 dev.o-m.kr에 배포되고 있습니다.

현재 주요 작업은 메인 랜딩 페이지의 아티클 목록 디자인 및 구현 단계입니다.

2. 기술 아키텍처 (Technical Stack)

AI 에이전트는 다음 기술 스택과 버전을 준수해야 합니다.

프레임워크: Astro 5.x (Static Site Generation 기반)

런타임 환경: Node.js

언어: JavaScript 기반. 필요 시 Typescript를 사용할 수 있음

스타일링:

Tailwind CSS v4

FlyonUI (Tailwind 플러그인 기반 컴포넌트)

데이터 시각화: D3.js v7 (필요 시)

패키지 매니저: npm

3. 디자인 시스템 및 테마 (Design System)

프로젝트는 "Paper-like(종이 질감)" 미학을 추구하며, 다음 규칙을 따릅니다.

3.1. 색상 팔레트 (Color Palette)

Paper Scale: 자체 정의된 oklch 기반 색상 스케일을 사용합니다.

제약 사항: 코드 내에서 #111111과 같은 하드코딩된 Hex 색상 값 사용을 엄격히 금지합니다. 반드시 아래의 CSS 변수 또는 이에 매핑된 Tailwind 클래스를 사용해야 합니다.

범위: --color-paper-50 (가장 밝은 배경) ~ --color-paper-950 (가장 어두운 텍스트)

3.2. 타이포그래피 (Typography)

목적에 따라 폰트 패밀리를 구분하여 사용합니다.

제목 및 UI 요소 (Headings & UI): Sans-serif 폰트 사용 (Tailwind 기본 font-sans 계열).

본문 및 읽기 텍스트 (Body & Reading): Serif 폰트 사용 (Tailwind font-serif 계열, 예: Georgia).

4. 데이터 아키텍처 (Data Architecture)

4.1. 콘텐츠 소스

모든 아티클 데이터는 Astro의 Content Collections (src/content/articles)를 통해 관리됩니다.

데이터 접근은 반드시 astro:content의 getCollection 함수를 사용해야 합니다.

4.2. 핵심 데이터 스키마 (Article Schema)

아티클 객체(article.data)는 다음과 같은 핵심 필드를 포함합니다.

headline (string): 기사 제목.

rubric (string): 기사 요약 또는 부제.

slug (string): URL 식별자.

date (Date): 발행일.

tags (string[]): 태그 목록.

revisions (Array<{ timestamp: string, ... }>): 수정 내역 배열. (정렬 로직의 기준이 됨)

5. 프로젝트 구조 및 핵심 파일 (Project Structure)

TDD와 책임 분리 원칙을 지원하기 위한 디렉토리 구조입니다.

/
├── .agent/                  # AI 에이전트 지침 폴더
│   ├── project_context.md   # (본 파일) 프로젝트 헌장
│   ├── tdd_rules.md         # TDD 워크플로우 규칙
│   ├── test_strategy.md     # 테스트 및 Mocking 전략
│   └── tasks/               # 현재 작업 명세서 폴더
├── public/                  # 정적 에셋
├── src/
│   ├── assets/              # 번들링 되는 에셋 (이미지, 스크립트 등)
│   ├── components/          # 재사용 가능한 UI 컴포넌트 (예: ListItem.astro)
│   ├── content/             # Astro Content Collections 정의
│   ├── layouts/             # 페이지 래퍼 (예: Paper.astro)
│   ├── pages/               # 라우트 및 페이지 진입점 (예: index.astro)
│   ├── styles/              # 전역 스타일 (theme.css 포함)
│   └── utils/               # [중요] 순수 함수 로직 (데이터 변환, 정렬기 등)
├── astro.config.mjs         # Astro 설정
├── tailwind.config.mjs      # Tailwind v4 설정 (FlyonUI 포함)
└── package.json




6. 개발 환경 및 배포

필수 조건: Antigravity 환경에서 AI 에이전트의 능동적 개입을 전제로 개발한다.

빌드 커맨드: npm run build (기본적으로 정적 사이트 생성)

개발 서버: npm run dev (로컬 개발용, 필요한 경우 HTTPS 지원)

## `.agent` 폴더의 관리

- `.agent` 폴더는 AI 에이전트의 지침을 담고 있는 폴더로, AI 에이전트가 프로젝트를 진행하는 동안 이 폴더의 내용을 참고하여 작업을 수행한다.
- `.agent\overview.md`는 언제나 최상위 폴더에 위치해야 한다.
- 사용자가 작성한 파일은 `.agent\usr\` 폴더에 추가한다.
